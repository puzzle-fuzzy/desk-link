# Windows 0.1.86：高 DPI 原始像素与 1:1 浏览

DeskLink 0.1.86 将视频协议版本提升到 10，并把 Windows 清晰档的输出上限扩展到 2560 × 1440。目标电脑为 2K 或更高分辨率时，主机不再强制先压成 1920 × 1080；在控制端使用“1:1 原始像素”可以检查收到的原始输出，画面超出视口时仍可滚动浏览。

## 兼容边界

- 两台 Windows 电脑必须同时升级到 0.1.86；协议版本 10 会让旧版本在目录阶段明确提示不兼容，不会进入黑屏重试循环。
- 公网中继不解析协议内容，不需要更新服务器。
- “均衡”和“流畅”仍保持 1920 × 1080 / 1280 × 720 上限，弱网或控制端压力下自动模式可以回到这些档位。

## 为什么这比继续加码率有效

如果被控电脑是 2560 × 1440，而视频仍先缩到 1920 × 1080，小字号文字已经在主机端丢失了像素信息；控制端再切换 1:1 也无法恢复。0.1.86 先保留清晰档的源分辨率，再结合 12 Mbps、峰值约束 VBR、CABAC 和高质量/速度档，减少这类不可逆的主机端损失。
