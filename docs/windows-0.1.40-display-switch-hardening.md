# Windows 0.1.40：多显示器切换与输入状态加固

## 用户结果

- 切换目标电脑的显示器前，控制端会清除仍处于按下状态的键、修饰键和鼠标按钮，避免切屏后遗留拖动或组合键；
- 屏幕切换超过 8 秒没有得到主机确认时，选择器会自动恢复，不会无限停留在禁用状态，也不会关闭当前远控会话；
- 主机拒绝或无法打开目标显示器时，工具栏会显示明确的中文结果，并继续保留原屏幕；
- 显示器位于主屏左侧、右侧、上方或下方，且分辨率不一致时，鼠标仍映射到当前远程屏幕的物理像素范围。

## 主机恢复边界

Windows 显示模式变化、显卡恢复或 DXGI 降级到 GDI 后，主机会重新读取完整虚拟桌面坐标并按当前采集尺寸重建 H.264 编码器。每个输入事件在注入前都会读取最新的已选桌面映射，避免视频已经恢复而鼠标仍使用旧坐标。

普通字符键现在也进入主机的按下状态记录。如果会话恰好在字符键按下与抬起之间断开，主机销毁输入边界时会补发对应的抬起事件，降低远端按键卡住的风险。

Windows 的 DXGI 桌面坐标采用物理像素；浏览器端先按实际画布归一化，再由主机映射到当前显示器和整个虚拟桌面，因此不使用控制端或被控端的 CSS 缩放比例推算坐标。

## 兼容与部署

本次没有增加协议消息，仍使用协议版本 4，不需要部署公网中继。0.1.40 可以与 0.1.39 建立会话；为了让控制端切屏超时恢复和主机坐标刷新同时生效，双机测试仍建议两台电脑都安装 0.1.40。
