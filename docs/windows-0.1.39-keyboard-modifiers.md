# Windows 0.1.39：修饰键与远程 Windows 键

## 用户结果

- `Ctrl`、`Alt`、`Shift` 和 Windows 键现在拥有独立的按下与抬起状态；
- 按住 `Shift` 或 `Ctrl` 再点击鼠标，可以在远程资源管理器等界面完成连续选择和多选；
- `Alt+Tab`、`Win+E`、`Ctrl+Shift+Esc` 等普通桌面组合键继续按真实按键顺序发送；
- 工具栏新增“Windows 键”，可打开远程开始菜单，不会触发本机开始菜单；
- 失焦、切换浏览模式、重新渲染或断开连接时，会先释放远端仍处于按下状态的普通键、修饰键和鼠标按钮。

Windows 安全桌面仍不接受普通应用注入，因此 `Ctrl+Alt+Delete` 仍必须在被控电脑本地操作。

## 状态加固

控制端按物理按键代码追踪状态。左右两个 `Shift` 或 `Ctrl` 同时按下时，只发送一次远端按下，并在最后一个物理键松开时才发送抬起。

普通组合键的抬起事件复用按下时记录的修饰状态。即使用户先松开 `Ctrl`、再松开字母键，也不会把远端 `Ctrl` 遗留在按下状态。批量释放按相反顺序处理普通键和修饰键，重复修饰键只释放一次。

## 协议边界

输入协议提升到版本 4，新增四种独立修饰键编码，并在协议、Windows WebView 边界、共享 FFI 与 Windows `SendInput` 边界重复校验。修饰键消息如果同时携带自身修饰位会被拒绝，避免重复注入。

协议升级不改变公网中继加入或转发格式，因此不需要部署中继。两台 Windows 电脑必须同时安装 0.1.39；0.1.39 不与仍使用协议版本 3 的 0.1.38 客户端建立业务会话。

## 自动验证

- Bun 测试覆盖修饰键映射、左右修饰键计数、组合键状态复用和反向释放；
- Rust 协议测试覆盖新按键往返和重复自身修饰位拒绝；
- Windows 输入测试覆盖四种独立修饰键及重复状态拒绝；
- 共享 FFI 测试覆盖新增键码 28 至 31 的双向转换。
