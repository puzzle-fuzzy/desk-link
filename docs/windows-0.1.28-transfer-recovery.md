# DeskLink 0.1.28 传输恢复说明

## 本版目标

0.1.28 在 0.1.27 剪贴板和文件通道的基础上补齐网络波动后的恢复体验，不改变中继协议和文件安全边界。

## 用户可见变化

- 文件传输状态同时显示已发送大小和文件总大小。
- 发送失败、用户取消或连接中断后显示“重新发送”。
- 重试会重新读取原文件、生成新的传输 ID，并再次要求被控端确认。
- 最近文件路径只保留在当前 DeskLink 进程内，不写入设置、日志或 WebView。
- 连接恢复不会自动发送文件，避免用户未确认的后台外传。

## 建议人工体验

1. 正常发送文件，确认大小与进度持续更新。
2. 发送中断开网络，等待远程控制恢复后点击“重新发送”。
3. 确认被控端再次弹出接收确认，并且完整文件只在校验成功后出现在“下载”文件夹。
4. 在重试前移动、删除或修改原文件，DeskLink 应重新检查文件并给出明确错误，不发送旧缓存内容。
5. 点击取消后再点击“重新发送”，确认不会复用上一轮传输 ID，也不会留下未完成文件。
