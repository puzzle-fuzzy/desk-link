# Windows 0.1.34：传输界面与远程控制隔离

DeskLink 0.1.34 减少大文件传输期间的前端主线程工作，让视频解码、画面绘制和输入处理继续保持优先。

## 行为变化

- 连续发送或接收进度最多每 100 毫秒绘制一次；
- 等待确认、开始传输、校验、完成、失败、拒绝和取消仍然立即显示；
- 队列与最近记录只在对应数据变化时重建，不再随每个文件块重复生成；
- 传输面板关闭后，不再更新隐藏面板中的进度 DOM；
- 工具栏“传输”按钮用小圆点表示正在传输，用数字表示未查看的完成或失败结果；
- 打开传输面板后，未查看结果立即标记为已读。

## 生命周期边界

- 页面重新渲染、连接中断或面板关闭时会取消尚未执行的进度刷新定时器；
- 新进度到达时始终覆盖内存中的最新状态，限频只影响界面绘制，不丢弃传输协议消息；
- 终态到达时会取消等待中的进度绘制并立即更新，避免完成后仍显示旧百分比；
- 相同终态信号不会重复增加未读计数，计数上限为 99。

本版本只优化 Windows 控制端界面，不改变端到端加密、文件内容、协议或公网中继。
