# Windows 视频直连通道设计与实现状态

## 目标

在不改变现有控制、审批、输入和文件传输链路的前提下，为远程画面增加一条可选的局域网直连视频通道。公网中继继续作为默认和故障回退路径；直连失败时，用户不应看到会话被“踢出”或进入无限重连。

## 当前状态

- `QuicClient` 继续承载 DeskLink 公网中继；Windows runtime 另外创建短生命周期 `DirectLanEndpoint`，只把视频数据报切到经过认证的局域网连接。
- `VideoPathKind`、`VideoPathQuality` 和 `VideoPathDecision` 提供路径分类、丢包测量和 4K 门控；当前探测只提供 RTT，丢包未测量时不会放行 4K。
- `DirectLanCandidate` 和 Candidate Offer/Answer 已经进入协议 12：只接受私有/回环地址、非零端口、非零会话绑定和最长 10 秒的有效期；Answer 同时携带回答方候选，双方可以正确探测对方端点。
- 会话绑定现在从每次 Noise 会话密钥独立派生 16 字节摘要；同一会话两端一致，不同连接自动变化，不会把设备对或旧连接当成永久信任。
- `DirectVideoPathMachine` 已实现候选 Offer、Answer、探测成功、探测失败、超时和停止回落；Windows runtime 已接入临时端点和认证探测连接，非 Windows 仍保持中继。
- 控制端已把候选消息接入认证后的控制事件循环；Windows 控制端会接受经会话绑定校验的入站直连，并在连接关闭时自动回到中继，不会因为直连失败重启控制会话。
- Windows 控制端和主机端在能力协商完成后都会发布短生命周期候选；主机端收到控制端候选后建立直连 QUIC 视频数据通道，控制、输入、审批、剪切板和文件继续留在可靠的加密中继通道。
- `desklink-transport` 已提供基于操作系统路由选择的本地私有地址发现与候选构造；它不发送应用数据，候选仍经过协议层地址、端口、有效期和会话绑定校验。
- 视频发送循环已经改用 `VideoDatagramBackend` 抽象；`RelayVideoPath` 保持原有中继发送，`DirectLanVideoPath` 可以复用已认证的直连 QUIC 数据报，不改 H.264 分片、关键帧恢复或控制链路。
- `DirectLanEndpoint` 已实现临时 QUIC 监听、候选校验和加密探测请求/响应；探测帧复用 `SecureSession` 的 `VideoDatagram` lane，校验会话绑定、候选 ID、随机 nonce 与时钟窗口。`connect`/`accept_probe_connection` 会在探测成功后保留连接，并由 Windows runtime 复用发送/接收该连接。
- `DirectLanSession` 将临时端口、本地候选和会话绑定放在同一生命周期对象中；端点销毁时候选不再可用，避免 runtime 把过期端口继续发布给对端。
- `QuicClient::remote_address` 和 `DirectLanSession::bind_for_client` 已提供安全的路由提示入口：它们只用于让操作系统选择本地网卡，不会把公网中继地址伪装成局域网候选。
- 当前开发协议已统一提升到 12，候选 Offer/Answer 直接属于控制协议；不再维护独立的旧版本直连信封。
- 直连只承载视频数据报，控制端和主机端均保留中继回落；路径指标会在直连接管后记录为 `directLan`，连接失效后恢复为 `relay`，4K 实验档仍不会因“有候选”而自动放行。
- 控制端诊断已经记录画面宽高、实际 H.264 Profile 和当前 `video_path`，可以直接比较中继与直连的清晰度、帧率和恢复次数。

## 直连不能这样实现

不能把中继地址替换成对端上报的 IP，也不能因为两个地址处于同一网段就跳过身份校验。这样会引入 DNS rebinding、局域网主动探测、会话劫持和旧候选重放风险。

## 计划中的安全流程

1. 两端先完成现有 Noise 会话和主机确认；视频直连候选只在这条已认证的控制通道中交换。
2. 候选包含短时有效期、随机候选 ID、端点、会话绑定摘要和对端身份签名。候选过期或会话切换后立即失效。
3. 控制端与主机分别对候选发起 QUIC 探测；探测握手必须绑定 Noise 会话摘要和双方设备身份，不能只依赖 TLS 证书。
4. 只有双方都确认同一个候选 ID，且探测 RTT、丢包率在门控范围内，才把 `VideoDatagram` 切换到 `DirectLan`。控制、输入、审批、剪切板和文件继续留在可靠的加密中继通道。
5. 直连在握手超时、身份绑定失败、连续丢包或路径改变时立即关闭，视频回落到 Relay；回落期间保留当前控制会话和最后一张画面。
6. 4K 仅在 `DirectLan` 通过 50 ms RTT、0.50% 丢包、解码队列和首个关键帧门槛后才允许，并且仍需双方声明 High Profile 和 3840×2160 能力。
