# Windows 0.1.50：文件恢复任务绑定目标设备

0.1.50 加固 0.1.49 的跨重启文件恢复，解决用户连接另一台电脑后误重试旧任务的风险。

## 目标设备绑定

- 每条恢复记录都保存原目标的完整 128 位设备身份，并继续使用当前 Windows 账户的 DPAPI 加密；
- 重试前必须已经建立远程控制连接，并且当前主机身份与恢复记录完全一致；
- 界面只显示便于识别的 12 位公开设备 ID，不把完整身份或文件路径交给 WebView；
- 连接其他设备时不会自动展开旧任务，后端即使收到重试命令也会拒绝发送并提示正确设备 ID；
- 0.1.49 生成的无目标绑定记录无法安全迁移，0.1.50 首次启动会失败关闭并清理它。

## 放弃恢复

文件传输失败或取消后，用户可以选择“重新发送/重新获取”，也可以点击“不再重试”：

- 发送任务只清除恢复元数据，绝不删除用户选择的源文件；
- 接收任务会按传输 ID 删除对应的隐藏断点，不扫描或删除其他任务；
- 暂存目录或目标文件若为符号链接、非普通文件或无法访问，操作会失败并保留恢复入口；
- 内存状态、DPAPI 文件和清理操作串行执行，避免新传输与放弃操作并发时状态错位。

## 协议与部署

目标绑定只修改控制端本地加密状态，不改变应用协议或线上 relay。协议版本保持 8，中继无需重新部署。
