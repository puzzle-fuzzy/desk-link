# Windows 0.1.59：有界视频 IPC

## 用户体验结果

0.1.59 解决控制端 WebView2 短暂忙碌时，完整 H.264 帧可能继续进入 Tauri 内部队列并积压的问题。远程桌面操作方式、画质选择、显示器切换和协议均不改变。

新交付边界固定为：

- WebView2 同一时刻只有一个视频拉取请求；
- Rust 端只保留一个等待交付的视频帧；
- 流 ID 或画面配置变化后，旧请求不能取得新画面；
- 断线、取消、重连和切屏会唤醒并结束旧请求；
- 邮箱满时不阻塞输入、音频、剪贴板或文件传输。

## 安全恢复

关键帧可以独立建立新的 H.264 参考链，增量帧则依赖之前画面。单槽已满时：

- 新关键帧替换槽内旧帧，优先保留更新的独立恢复点；
- 新增量帧被丢弃，当前参考链关闭，并通过已有加密控制通道请求关键帧；
- 后续增量帧在关键帧到达前不会交给 WebCodecs；
- 关键帧请求最多每秒一次，持续溢出会向自动画质合并有界播放压力。

网络重组边界与 Windows IPC 边界现在共用 `desklink-video::VideoContinuity`，避免两套恢复规则逐渐偏离。
