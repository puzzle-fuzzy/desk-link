# Windows 0.1.32：文件队列、拖放与最近记录

DeskLink 0.1.32 在双向文件传输基础上增加了批量发送体验。

## 新增功能

- Windows 文件选择器支持一次选择多个文件；
- 连接成功后，可以把一个或多个文件拖到远程画面；
- 发送队列最多保留 20 个文件，并严格逐个传输；
- 等待中的文件可以单独移除，也可以一次清空；
- 失败、拒绝或取消当前文件时，后续队列会暂停；
- 用户可以重新发送失败文件，或跳过当前结果继续队列；
- 传输面板显示最近 8 条完成、失败、拒绝或取消记录。

最近记录只保存在当前应用运行期间，不会把文件名写入新的持久化存储。

## 稳定性边界

- 底层仍然同时只传输一个文件；
- 文件分块、大小限制、临时文件和哈希校验保持不变；
- 文件传输继续使用独立的端到端加密可靠通道；
- 文件排队不会进入鼠标、键盘、视频或音频通道；
- 单个文件最大 256 MB，队列最多 20 项；
- 拖入文件夹、无效文件或超限文件时，整个新增操作会失败，不会留下部分队列。

该版本不需要重新部署 DeskLink 公网中继。
