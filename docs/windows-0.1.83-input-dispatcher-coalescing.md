# Windows 0.1.83：输入分发器移动合并

## 目标

在 Tauri 输入命令尚未完成时，避免高轮询率鼠标为每次动画帧创建新的队列项，同时保留离散输入的严格顺序。

## 改动

- `RemoteInputDispatcher.enqueueMouseMove` 使用一个可复用待发送槽保存最新鼠标坐标。
- 输入命令繁忙时，连续移动只覆盖槽内坐标；发送时才创建一次 `ControllerInput`。
- 键盘、鼠标按键和滚轮到达时，会先刷新待发送移动，确保控制顺序不变。
- 断开、切屏和清理待处理移动时会同步清空该槽；不同 stream 的移动不会合并。

## 验证

- 输入分发器新增忙碌期间复用和离散输入顺序测试。
- 需要在两台 Windows 实机上观察快速移动、拖拽、按键组合、切屏和重连时的输入顺序与跟手性。
