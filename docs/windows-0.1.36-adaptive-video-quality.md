# Windows 0.1.36：自适应远程画质

DeskLink 0.1.36 在现有“流畅、均衡、清晰”手动档位之外增加“自动”模式。控制端每秒汇总一次视频分片的增量收包与缺失数量，通过端到端加密控制通道反馈给目标主机；中继只能转发密文，不能读取网络指标或画质选择。

## 调整策略

- 少于 40 个视频分片的稀疏窗口不参与判断，避免静止画面造成误判；
- 连续两个窗口的缺失率达到 5% 才逐级降档，单个窗口达到 15% 时立即逐级降档；
- 只有连续 12 个窗口的缺失率不高于 0.5% 才逐级升档；
- 每次切换后保留 8 个有效窗口的冷却期，避免网络抖动时反复重建编码器；
- 自动模式只在“清晰 → 均衡 → 流畅”之间逐级移动，不会跨档跳变；
- 用户选择任一手动档位后立即停止自动判断，并锁定该档位。

画质变更仍然只重建目标主机的 Media Foundation H.264 编码器。Noise 会话、QUIC 连接、主机批准状态、输入通道、剪贴板和文件传输均保持不变。编码器无法应用新档位时继续使用原档位，不会因此中断远程控制。

## 丢包统计修正

此前控制端的“丢包”计数主要覆盖无效、过期或重复分片，无法完整反映不完整视频帧中实际缺少的分片。0.1.36 会在帧超时和重组容量淘汰时统计缺失分片数量；视频流版本切换会清空旧流的未完成帧，不把正常的编码器重建误判为网络丢包。

## 兼容性

此次扩展将端到端协议版本提升到 3。两台 Windows 电脑必须都安装 0.1.36；旧客户端会在 Noise 握手阶段明确判定版本不兼容，不会进入不稳定的半连接状态。公网中继不解析这些业务消息，因此不需要随此版本重新部署。

为避免改变已有用户的画质选择，升级后默认仍为“清晰”手动档。需要由 DeskLink 根据网络状况调整时，在远控工具栏的画质选择器中切换为“自动”；选项会同时显示当前实际档位，例如“自动（均衡）”。
