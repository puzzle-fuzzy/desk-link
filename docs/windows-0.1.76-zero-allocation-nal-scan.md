# Windows 0.1.76 无分配 NAL 类型扫描

控制端收到视频包后，需要确认传输层没有标记的关键帧是否实际包含 H.264 IDR。此前这条路径调用完整 `annexBNalTypes()`，为每个检查创建类型数组；在持续视频流中会制造不必要的短生命周期对象。

## 新路径

- `isH264Keyframe()` 改用 `hasAnnexBNalType()`，直接扫描 Annex B 起始码和 NAL 类型，找到目标类型后立即返回。
- `annexBNalTypes()` 仍保留给关键帧参数拼接等确实需要完整类型列表的低频路径。
- 输入格式、IDR 识别规则、关键帧恢复和协议 9 均不改变。

## 验证

- 单测覆盖三字节/四字节起始码、IDR、非 IDR、显式关键帧标记和参数集拼接。
- 生产构建通过；真实 Windows 测试重点观察长时间 30 FPS 会话的 CPU、GC 抖动和关键帧恢复行为。
