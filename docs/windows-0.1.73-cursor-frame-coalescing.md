# Windows 0.1.73 远端光标帧合并

远端光标是独立于视频帧的实时信号。旧路径每收到一个坐标就立即写入 DOM `transform`；当主机高频发送光标、控制端同时进行 Canvas 绘制或窗口缩放时，这些样式提交会争用 WebView2 主线程。

## 新路径

- `LatestFrameScheduler` 在一个动画帧内只保留最后一个光标值。
- 每个动画帧最多执行一次坐标换算和 `transform` 写入，光标不会因为排队而落后于最新信号。
- 断开、重连、切换远端显示器或重建远程画面时，取消待提交帧并丢弃旧坐标。
- 该优化只改变本地显示提交节奏，不改变光标协议、坐标单位或输入事件顺序。

## 验证边界

- 单测覆盖突发值只提交最新坐标，以及取消后不再提交旧值。
- Canvas、输入队列和视频解码仍使用各自的节流/有界路径，互不共享可变队列。
- 真实双机测试时重点观察：远程光标是否跟随最新位置、窗口拖动/全屏切换是否残留旧光标、以及高频移动时视频帧率是否更稳定。
