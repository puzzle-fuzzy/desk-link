# Windows 0.1.77：最新视频帧队列

## 目标

Windows 主机端使用“只发送最新编码帧”的有界队列，以优先保证远程控制的实时性。此前发送循环通过 `drain_newest_first()` 把整个队列收集到临时 `Vec`，随后只取第一个元素；这会在每次视频发送唤醒时产生不必要的短生命周期分配。

## 本次调整

- `LatestFrameQueue::take_newest()` 先从队尾取出最新帧，再原地清空过期帧；队列容量和丢帧策略不变。
- 主机发送循环使用 `take_newest()`，不再为已丢弃的帧构造中间数组。
- 显示器切换、画质切换和采集恢复使用无分配 `clear()`，避免恢复边界额外触发分配。
- 保留 `drain_newest_first()` 作为通用有序排空 API，未改变共享 crate 的既有行为。

## 影响边界

该改动只影响主机进程的内存分配路径，不改变协议 9、QUIC 数据报格式、帧号、关键帧恢复或控制端解码行为。队列仍最多保留配置的两帧；发送端忙碌时，旧帧会被丢弃，下一次发送继续使用最新帧。

## 验证

自动验证覆盖队列的“取最新并清空”和“无分配清空”语义，以及现有视频分片、Rust、前端测试和 Windows 发布门禁。两台真实 Windows 电脑的长时间帧率、CPU/GC、网络抖动和多显示器体验仍需人工验收。
