# Windows 0.1.52：等待队列保护状态与失败恢复

0.1.52 加固 0.1.51 的跨重启等待队列，重点解决本地加密存储失败时界面无法及时说明风险，以及会话重建时相同队列被重复写入的问题。

## 用户可见行为

- 有等待文件且 DPAPI 记录写入成功时，传输面板明确显示“等待队列已由当前 Windows 账户加密保存”；
- 写入失败时，运行中的队列仍可继续使用，但界面立即显示“仅保留到本次运行”和准确原因，并提供“重试保护”；
- 即使队列内容没有变化，只要上一次保护失败，重试仍会再次写入，不会被去重逻辑跳过；
- 清空队列的加密记录删除失败时，后端恢复原内存记录并返回错误，界面保留恢复入口，不会误报为已清空；
- 保护或清理操作进行时，相关继续、移除、清空和重试操作暂时禁用，避免并发动作覆盖新状态。

## 写入与一致性

- 目标设备、暂停状态、文件顺序、传输 ID、名称、大小、路径和修改时间均一致时，不重复执行 DPAPI 原子写入，也不递增恢复 revision；
- 队列真实变化仍同步完成 DPAPI 加密和原子替换后再发布状态，保留崩溃后的强一致恢复语义；
- 当前写入只发生在添加、移除、暂停、继续和完成等队列变更点，不位于视频、鼠标或音频实时数据路径，因此本阶段不引入异步写后确认模型。

## 验证边界

新增测试覆盖成功保护、重复状态去重、写入失败后的同状态重试、清除失败保留记录，以及中文界面的成功、降级和忙碌状态。队列恢复仍只修改 Windows 本地状态与界面；DeskLink 协议版本保持 8，公网中继无需重新部署。
