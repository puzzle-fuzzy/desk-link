# Windows 0.1.41：重连输入隔离与队列延迟加固

## 用户结果

- 远控连接中断后，尚未发送的旧鼠标和键盘事件会立即清除，不会在重连成功后突然作用到远端；
- 浏览器端仍按下的普通键、修饰键和鼠标按钮会在离开已连接状态时清空，新的远程画面从干净输入状态开始；
- 网络短时阻塞时，DeskLink 限制 Rust 队列中可替代的旧鼠标移动数量，减少连接恢复后的鼠标追赶和卡顿；
- 鼠标按钮和键盘的按下、抬起仍使用可靠有序路径，队列压力不会静默丢弃关键释放事件。

## 会话隔离边界

控制端 WebView 把每个实时输入与当前远程视频流 ID 一起加入本地队列。连续鼠标移动只有在属于同一流时才允许合并；断线后会清空等待队列。

Tauri/Rust 边界会在输入进入命令队列前核对活动流，并在真正发送前再次核对。即使一个旧的 IPC 请求跨越了断线和重连，它携带的旧流 ID 也会被忽略，不可能借用新连接的身份发送。

Rust 共享命令队列最多允许少量待处理鼠标移动；达到限额后只丢弃新的可替代位置，并为键盘、鼠标按钮、切屏、画质和恢复命令保留空间。丢弃计数继续进入脱敏诊断，用于识别真实网络或本机负载问题。

## 兼容与部署

本次只修改 Windows WebView 与本机 Rust 命令边界，没有改变端到端协议，仍使用协议版本 4，也不需要部署公网中继。0.1.41 可以连接 0.1.40 和 0.1.39；由哪台电脑发起控制，哪台电脑安装 0.1.41 就能获得输入隔离，但双机仍建议保持同一版本。
