# Windows 0.1.67 渲染性能诊断

0.1.67 在控制端记录两个仅用于诊断的渲染指标：

- `displayed_fps_x100`：从首帧到当前帧的平均显示帧率，放大 100 倍后记录，避免浮点格式和跨语言误差。
- `max_frame_gap_ms`：本地 Canvas 已显示帧之间的最大间隔，用来识别卡顿尖峰。

指标在每次视频诊断周期上报，不改变默认页面，也不把帧级数据发送到中继。帧回调只更新原地计数器，快照和 IPC 调用仍受原有诊断节流限制。

排查时可按以下顺序判断：

1. 中继完成帧持续增长但显示帧率低：优先检查本机解码或 WebView2。
2. 显示帧率正常但最大帧间隔很大：优先检查渲染线程短时阻塞。
3. 中继完成帧和收到帧都停滞：优先检查网络、中继会话或目标主机编码。
