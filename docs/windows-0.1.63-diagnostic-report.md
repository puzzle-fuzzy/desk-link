# Windows 0.1.63：诊断报告完整性

## 本阶段目标

性能由两台 Windows 电脑手动测量；代码侧补齐故障证据链。此前“导出诊断报告”只读取主机日志，控制端的中继重试、视频邮箱压力、WebView2 拉取失败和解码恢复记录仍留在另一份本地日志中，容易让报告缺少真正的控制端证据。

## 已完成

- 诊断报告同时读取主机 host.jsonl 与控制端 controller.jsonl。
- 两类事件按 timestamp_unix_ms 合并排序，最多保留最近 200 条，并分别标记为 [主机] 或 [控制端]。
- 保持原有敏感信息清理和报告长度上限；设备完整身份、会话密钥、访问密码、长十六进制值和文件内容不会进入报告。
- 不新增网络协议或中继字段；云端诊断上传仍按既有来源分别脱敏上传。

## 验证

- 新增日志合并与时间排序单测。
- 既有报告脱敏单测继续验证中继地址可诊断、身份密钥不可见。
- Rust、UI、发布构建继续执行既有门禁。
