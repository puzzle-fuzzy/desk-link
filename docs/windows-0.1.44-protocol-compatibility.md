# Windows 0.1.44：连接协议兼容性预检

## 问题

应用协议版本不一致时，旧端可能在 Noise 握手解码失败后先关闭传输。另一端只能观察到连接中断，容易进入无意义的重试并把版本问题误报为网络故障。

## 处理方式

- host 在 relay 目录登记中附带非秘密的 `PROTOCOL_VERSION`。
- controller 在设备目录查询中附带自己的协议版本。
- relay 先使用恒定时间比较验证访问密码。密码不匹配时仍统一返回设备不可用，不暴露设备是否在线、host 版本或是否存在版本差异。
- 只有密码匹配后，relay 才比较双方协议版本。版本不一致时返回专用状态和 host 协议版本，不下发连接邀请。
- controller 根据版本大小提示“当前控制端版本过旧”或“目标电脑版本过旧”，并把结果标记为不可重试。
- 握手层仍保留 `ProtocolError::UnsupportedVersion` 的兜底分类，避免绕过目录的连接路径重新进入重试。

## 滚动部署

relay 同时接受旧版目录查询信封、旧版目录登记信封和新版带协议字段的信封。旧查询与旧登记之间保持原行为；新旧信封交叉时返回版本不兼容。部署顺序必须是 relay 先于 Windows 0.1.44 客户端。

## 自动验证

- transport 单测覆盖新版信封往返、协议版本为零时拒绝、旧版目录查询与 v3 登记信封继续解码。
- relay 集成测试覆盖匹配密码下的版本拒绝，以及错误密码加版本差异仍只返回设备不可用。
- Windows 控制端测试覆盖升级方向文案、不可重试分类和握手解码兜底。
