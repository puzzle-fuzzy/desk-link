# DeskLink 0.1.26 Windows 双机验收

这份清单只包含必须在两台真实 Windows 电脑上确认的行为。不要记录访问密码、完整设备身份或远程画面中的隐私内容。

## 准备

- 两台电脑都安装 `DeskLinkSetup-0.1.26-x64.exe`，并在“关于”确认版本为 0.1.26。
- 两端“设置”中开启“共享脱敏诊断”；如果出现问题，点击“立即发送诊断”并记下发生时间。
- 先在同一网络测试，再让其中一台切换手机热点，覆盖不同公网网络。

## 必须通过

1. 使用临时密码连接时，被控端出现确认窗口；拒绝后控制端结束，批准后能看到画面并控制鼠标、键盘。
2. 连续快速移动鼠标并滚动两分钟，被控端不能变为“主机服务已停止”，控制端不能进入“已有另一台连接”的重连循环。
3. 鼠标持续移动期间点击“断开连接”，界面应及时返回设备列表；再次连接同一保存设备应成功。
4. 连接后等待首帧；如果硬件解码不可用，软件应自动切换到兼容解码模式，不应长期停留在黑屏启动提示。
5. 画面正常后持续控制 30 分钟，包含窗口拖动、视频播放和十分钟连续鼠标活动；画面中途冻结时应自动请求关键帧并重建解码器，不能断开整个控制会话。
6. 在 100%、125% 和 150% Windows 缩放下点击画面四角与中心，远端鼠标位置应一致。
7. 被控端有两个显示器时，在两个屏幕之间来回切换五次；每次画面和鼠标坐标都对应当前选中屏幕。
8. 控制中断网十秒后恢复，只允许一个重连流程；主机保持可连接，不能在“已在线”和“已中断”之间持续闪烁。

## 结果记录

- 每项记录“通过”或“失败”，失败时附发生时间、两端状态文字、网络类型、显示器布局和复现步骤。
- 失败后点击一次“立即发送诊断”。0.1.26 会分别上报传输完成帧、前端收到帧、实际显示帧、首帧耗时、解码恢复次数和输入队列等待次数；这些都是脱敏计数，不上传屏幕或输入内容。
