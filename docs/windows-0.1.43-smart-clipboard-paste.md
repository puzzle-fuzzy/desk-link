# Windows 0.1.43：远程剪贴板智能粘贴

## 用户路径

远程桌面画面获得焦点后，用户可以直接按 `Ctrl+V`。DeskLink 会读取控制端当前纯文本剪贴板，并把内容粘贴到远端当前获得焦点的应用中。`Ctrl+Shift+V` 同样按纯文本处理；Windows `AltGr` 组合键不会被误判为粘贴。

快捷粘贴继续使用现有 48 KB 纯文本上限。不支持图片、富文本或后台自动同步，剪贴板只有在用户明确按下快捷键或点击传输按钮时才会被读取。

## 顺序与失败边界

1. WebView 拦截本次 `Ctrl+V`，丢弃尚未发送的鼠标移动，释放已按下的远端键和鼠标按钮，并等待有界输入队列排空。
2. Rust 边界读取 Windows 剪贴板，通过独立的端到端加密传输消息写入远端剪贴板。
3. 控制端用请求 ID 关联远端结果。只有匹配请求返回 `Completed` 后，才向远端发送带 Control 修饰键的 V 按下与松开事件。
4. 读取失败、内容不受支持、超过上限或远端拒绝时，不发送粘贴按键；传输面板显示中文原因，远程会话和视频流保持不变。

这一实现避免逐字发送长文本，也避免在远端剪贴板尚未写入时提前粘贴。快捷粘贴等待确认期间会抑制重复键盘输入，连接断开时自动清理等待状态。

## 自动验证

- 纯 TypeScript 测试覆盖 `Ctrl+V`、`Ctrl+Shift+V`、不同键盘布局、按键重复、AltGr、Meta 与无关快捷键。
- Rust 测试固定远端粘贴事件必须包含一对带 Control 修饰键的 V 按下/松开。
- 前端 TypeScript/Vite 构建、Rust Clippy 和 workspace 测试继续作为 Windows 发布门禁。
