# DeskLink 0.1.27 剪贴板与文件传输验收

## 功能边界

- 剪贴板只传输 UTF-8 纯文本，单次最多 48 KB，不自动监听或同步。
- 文件由控制端主动选择，被控端必须在 Windows 原生确认框中允许，单个文件最多 256 MB。
- 文件内容使用独立可靠通道并保持端到端加密，不与鼠标、键盘、控制消息或视频共用队列。
- 接收端只接受安全文件名，先写入 `%LOCALAPPDATA%\DeskLink\Transfers` 临时文件，完整性校验成功后移动到“下载”文件夹。
- 同名文件自动添加数字后缀，不覆盖已有文件；失败、拒绝、取消或断线时删除未完成的临时文件。

## 双机手动验收

1. 两台电脑均安装 `DeskLinkSetup-0.1.27-x64.exe`，完成正常远程控制。
2. 在控制窗口点击“传输”，分别验证“发送本机剪贴板”和“读取远端剪贴板”。中文、换行和符号应完整保留。
3. 发送一个小文件，被控端拒绝：控制端应显示“远端已拒绝”，下载文件夹中不应出现文件。
4. 再次发送并允许：进度应从等待确认进入发送、校验、完成，文件应出现在被控端“下载”文件夹且内容一致。
5. 再发送同名文件：原文件不得被覆盖，新文件名应带 `(1)` 等数字后缀。
6. 发送过程中点击取消：控制会话和鼠标键盘应继续工作，接收端不应留下可见的未完成文件。
7. 发送过程中持续移动鼠标并输入文字：远程输入不应因文件通道拥塞而停顿或断开。
8. 断开网络后恢复：未完成文件应报告失败；远程会话恢复后可重新选择文件发送。

## 安全负面验证

- 纯文本超过 48 KB 应在发送前被拒绝。
- 文件超过 256 MB 应在发送前被拒绝。
- 接收端不得接受路径分隔符、Windows 保留名或任意远程落盘路径。
- 校验失败的文件不得移动到“下载”文件夹，也不得执行或自动打开。
