# Windows 0.1.45：可靠文件接收

## 用户结果

- Windows“下载”目录即使被迁移到其他磁盘，接收完成后也能正常发布文件；
- 远端拒绝、正在进行另一项传输、下载目录不可写或磁盘读写失败会显示不同的中文原因；
- 连接中断、取消或校验失败仍会删除本次未完成内容；
- 新接收开始前会清理超过 24 小时的 DeskLink 崩溃遗留分片，不扫描或删除其他文件。

## 存储边界

接收数据先写入“下载”目录下的隐藏 `.desklink-transfers` 暂存目录。暂存文件和最终文件因此始终位于同一卷；BLAKE2s-256 内容校验、文件落盘同步完成后，才通过同卷重命名发布到最终文件名。已有同名文件不会被覆盖。

清理逻辑每次最多检查 256 个目录项，只识别 `.incoming-<32 位十六进制传输 ID>.part`，并且只删除修改时间超过 24 小时的普通文件。符号链接、无关文件和仍可能处于活动状态的新分片不会被清理。

## 协议兼容性

文件接收决定由布尔值升级为明确的 `TransferResult`，应用协议版本从 4 升到 5。`Completed` 表示接收端已准备好接收分片；其他结果说明拒绝、占用、大小、权限或读写失败。0.1.44 的目录版本预检会在连接前拦截混合版本，不会把不同消息布局交给会话解码器。

本版本不改变 relay 转发格式，因此不需要重新部署中继；发布时只需确保两台 Windows 电脑升级到相同版本。

## 自动验证

- 协议往返测试覆盖带原因的文件接收决定；
- Windows 存储测试覆盖同卷暂存路径、过期分片清理、无关文件保留、超大文件提前拒绝、内容哈希和同名文件保护；
- 控制端测试覆盖拒绝、占用和磁盘失败的中文呈现；
- 完整发布门禁继续运行格式、Clippy、Rust workspace、Bun、TypeScript/Vite 和 Windows x64 安装器构建。
