# DeskLink 公网中继运维基线

## 当前边界

- Windows 客户端只把中继当作会话目录和端到端密文转发层；中继不能读取屏幕、输入、访问密码或会话密钥。
- `turn.p2p.yxswy.com:4433/udp` 当前运行在单台腾讯云主机，外部 QUIC/TLS 探针每 30 分钟执行一次。
- 容器必须启用健康检查、`always` 或 `unless-stopped` 重启策略、只读根文件系统、非 root 用户和连接/会话上限。
- 中继每分钟只记录聚合容量：活动会话数、已附加参与方数、QUIC 连接数及上限；不记录设备 ID、访问密码或业务内容。

## 巡检

本机已加载 SSH 密钥后运行：

```text
python scripts/audit-managed-relay.py --target root@101.35.246.159 --identity-file C:\Users\18267\.ssh\p2p-tencent-ed25519
```

巡检会失败于以下任一条件：容器未运行或不健康、无自动重启策略、证书剩余不足 21 天、根分区使用率超过 80%、缺少容量样本、会话或连接容量达到 80%。报告写入 `dist/linux/managed-relay-host-audit.json`。

外部链路验证使用生产客户端完成 DNS 配置、UDP 4433、QUIC 与 TLS 身份校验：

```text
python scripts/verify-managed-relay.py
```

部署由脚本校验归档 SHA-256，保留旧镜像回滚标签，并在新容器达到健康状态前不判定成功：

```text
python scripts/deploy-managed-relay.py --target root@101.35.246.159 --identity-file C:\Users\18267\.ssh\p2p-tencent-ed25519
```

## 告警与处置

1. 外部探针失败：先确认腾讯云安全组和主机 UDP 4433，再检查容器健康、重启次数和最近日志。
2. 证书窗口不足：执行续期演练，确认容器读取的新证书后重建或重启中继，再运行外部探针。
3. 容量达到 80%：先提高实例规格或限制，再部署第二节点；不要直接移除准入上限。
4. 容器反复重启：停止自动发布，保留当前日志和镜像 ID，回滚到上一份已通过探针的镜像。

## 第二节点与切换

真正的自动容灾还需要第二台具有独立公网 IP 的主机。当前客户端配置保存的是 `SocketAddr`，不能仅通过 DNS 改记录完成无损切换；在第二节点存在前，不应声称已经具备自动故障转移。

后续实施顺序：

1. 在不同可用区部署同版本中继，使用同一受信任域名证书并单独执行探针。
2. 把客户端托管配置升级为“中继端点列表”，主机同时登记，控制端按优先级查找；任一端点失败时带抖动退避切换。
3. 会话目录保持无共享敏感数据；短期采用双登记，后续如需共享目录，只复制一次性邀请密文和过期时间。
4. 演练主节点断网、证书失效、容量耗尽和整机故障，并以双机远程画面恢复时间作为验收指标。
