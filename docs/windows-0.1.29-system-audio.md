# DeskLink 0.1.29 Windows 系统声音说明

## 本版目标

0.1.29 为 Windows 双机远程控制增加被控端系统输出声音。声音是独立、可降级的会话能力：声卡不可用、设备切换、单个音频包损坏或浏览器播放失败，都不得中断画面和控制。

## 用户可见变化

- 远程连接成功后默认播放被控电脑的系统输出。
- 远程工具栏新增 Lucide 声音按钮，可即时静音和恢复。
- 静音不仅把本机音量设为零，还会通知被控端停止发送声音，避免继续占用网络。
- 远端声音不可用时按钮显示“无声音”，远程控制继续工作。
- 重连后会恢复本轮控制会话中的静音选择。

## 技术边界

- 被控端使用 WASAPI loopback 采集当前默认渲染设备。
- 设备原始混音格式在本机下混并重采样为 48 kHz、单声道、16 位 PCM。
- 每包 10 毫秒、960 字节有效载荷，单包编码结果受 1120 字节协议上限约束。
- 音频使用独立 QUIC 数据报和独立 Noise AEAD lane，不进入视频、输入、控制或可靠文件队列。
- 主机队列容量为 8 包，积压时丢弃最旧声音，避免通过增加延迟掩盖网络拥塞。
- 控制端通过 Web Audio 以小幅前置缓冲调度；流切换、序列断裂、时间戳异常或排队超过 250 毫秒时清空旧调度。
- 当前不采集麦克风，不提供语音对讲，也不承诺受 DRM 保护内容可以被回环采集。

## 建议双机体验

1. 先部署支持音频数据报的新中继，再在两台 Windows 电脑安装 0.1.29。
2. 被控电脑播放普通网页视频或系统提示音，确认控制端能听到且画面/鼠标保持流畅。
3. 点击“静音”，确认声音立即停止；等待数秒后点击“打开声音”，确认从当前声音恢复而不是补播旧内容。
4. 在被控电脑切换默认输出设备，确认声音可短暂显示不可用并自行恢复，远控会话不重连。
5. 断网后等待会话恢复，确认声音和用户之前选择的静音状态一同恢复。
