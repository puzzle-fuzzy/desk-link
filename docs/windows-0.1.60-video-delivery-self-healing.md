# Windows 0.1.60：视频交付自愈与诊断

## 用户体验结果

0.1.60 修复控制端 WebView2 或 Tauri 视频拉取命令发生一次瞬时失败后，串行拉取循环可能永久结束、连接仍在但画面不再更新的问题。现有远程控制方式、双屏切换、画质选择、输入、音频、剪贴板和文件传输均不改变。

拉取失败后依次等待 100、250、500、1000、2000 毫秒再试，后续失败保持最多 2 秒退避；成功取得一帧后立即清除连续失败次数。断线、取消、切屏或视频配置变化仍会使旧请求和旧重试失效，不会把旧画面送进新会话，也不会产生第二个并发拉取请求。

## 可诊断性

Rust 视频邮箱新增三个累计指标：实际交付帧、邮箱溢出丢帧和关键帧替换。前端同时上报拉取失败次数，所有计数都使用饱和边界。启用“共享脱敏诊断”后，服务端可以进一步区分：

- 网络已经完成视频帧，但本机视频邮箱没有交付；
- 本机邮箱持续满载，控制端取帧速度暂时跟不上；
- WebView2 与 Rust 之间的视频拉取命令反复失败；
- 视频已经进入前端，但没有被解码或显示。

诊断只记录计数、耗时、阶段和脱敏原因，不记录画面、键盘内容、访问密码或完整设备身份。旧版本日志没有新增字段时继续按旧证据分析，不会被误判为视频 IPC 停滞。
