# Windows 0.1.33：传输速度与接收文件定位

DeskLink 0.1.33 继续完善双向文件传输的可理解性，不改变文件协议和中继职责。

## 新增体验

- 发送或接收文件时显示实时传输速度；
- 根据平滑后的速度显示预计剩余时间；
- 等待队列同时显示文件数量与总大小；
- 成功接收文件后显示“打开下载文件夹”；
- 打开目录的原生命令固定使用当前 Windows 账户的 Downloads 已知文件夹，不接受前端传入路径。

## 稳定性边界

- 速度采样至少间隔 200 毫秒，使用指数平滑减少短时跳动；
- 文件、方向、总大小变化，进度回退、系统时钟回退或控制连接中断时立即重置采样；
- 尚未形成有效速度、正在等待确认、正在校验或已经完成时不显示误导性的剩余时间；
- 极短传输可能只显示字节进度，不会为了显示速度而延迟完成；
- 队列仍然逐个传输，文件内容继续经过独立的端到端加密可靠通道与 Blake2s 完整性校验。

本版本仅修改 Windows 客户端，不需要重新部署 DeskLink 公网中继。
