# Windows 实时会话体验设计

## 目标

DeskLink 0.1.56 聚焦已经连接成功后的直接体感：普通用户不需要理解帧计数、丢包或手动画质，也能默认获得连续的 30 FPS 操控画面；进入全屏后，远程桌面应占据全部显示区域，工具栏只在用户需要时出现。

## 范围

本阶段包含：

- 默认使用自动画质，仍允许用户在当前会话中手动选择流畅、均衡或清晰；
- 流畅、均衡、清晰三个档位都使用 30 FPS，只通过分辨率和码率控制带宽；
- 主工具栏只显示分辨率、当前画质和端到端加密状态，不显示中继帧数、前端帧数、显示帧数或累计丢包率；
- 详细媒体指标继续写入现有脱敏诊断日志，不降低排障能力；
- 原生全屏中，工具栏在显示 3 秒后自动收起；鼠标移到屏幕顶部、键盘焦点进入工具栏、打开文字或传输面板时立即显示；
- 每次进入全屏时短暂显示“鼠标移到顶部显示工具栏 · Esc 退出全屏”，随后由同一闲置逻辑收起；
- 主动断开、意外断线和退出全屏继续释放远端输入状态并恢复普通窗口布局。

本阶段不包含：麦克风/语音对讲、整块虚拟桌面合成、自动下载安装更新、代码签名、中继第二节点、协议版本升级或 macOS 改动。

## 方案比较

### 方案 A：智能流畅与沉浸全屏（采用）

在现有 H.264、自动画质反馈和原生全屏基础上调整默认值与呈现。改动集中，能直接改善核心远控体感，不引入新网络消息。

### 方案 B：扩展文件与剪贴板类型

增加目录、图片剪贴板或拖放下载。价值明确，但现有纯文本和多文件队列已经满足基本传输需求，对“控制起来是否顺滑”的帮助较小。

### 方案 C：中继高可用与自动更新

增加第二中继节点、故障切换和后台升级。可靠性价值高，但依赖额外基础设施和代码签名，不适合作为当前本地代码阶段的第一优先级。

## 架构

### 主机画质策略

`apps/windows/src/runtime.rs` 保留现有 `AdaptiveVideoQuality` 状态机和每秒增量网络反馈。初始偏好由 `Sharp` 改为 `Automatic`，初始实际档位仍为 `Sharp`；连续两个退化样本后逐级下降，长健康窗口后逐级恢复，现有冷却窗口保持不变。

三个档位固定为：

- 流畅：1280×720 上限，30 FPS，1.5 Mbit/s；
- 均衡：1920×1080 上限，30 FPS，2.5 Mbit/s；
- 清晰：现有捕获上限，30 FPS，4 Mbit/s。

这样网络变差时优先牺牲空间细节，不主动把交互画面降到明显跳动的 15/20 FPS。码率与现有值相同，因此不会扩大公网带宽上限。

### 控制端状态呈现

新增纯 TypeScript 模块 `apps/windows-ui/src/remote-session-presentation.ts`，负责：

- 根据视频配置、用户偏好、实际档位和短时网络状态生成中文状态文案；
- 根据全屏、最近本地活动、顶部触发区、工具栏焦点和面板打开状态计算工具栏是否可见；
- 不接触 Tauri IPC、DOM、视频解码器或远程输入队列。

`controller.ts` 只负责把真实事件转换为该模块的输入，并用 `data-remote-toolbar-visible` 更新现有 DOM。顶部触发检测不会触发全页面重绘，也不会产生新的网络消息；只复用一个 3 秒计时器并直接更新属性。

### 全屏交互

非全屏始终显示工具栏。全屏进入时显示工具栏并启动 3 秒计时；工具栏自身的指针或键盘操作会延后收起。传输面板或文字输入打开时工具栏固定显示，关闭后重新计时。鼠标进入顶部 12 像素触发区时显示，离开后重新开始 3 秒计时。远程 Canvas 内的正常鼠标和键盘输入不会唤醒或固定工具栏，`Esc` 仍优先退出全屏。

工具栏隐藏只使用 180 ms 的位移与透明度状态过渡；`prefers-reduced-motion: reduce` 时关闭过渡。远程 Canvas 的尺寸、黑边 contain 计算和鼠标映射不改变。

## 错误与恢复

- 自动画质切换失败时沿用现有主机确认超时与可重试行为，不断开会话；
- 工具栏状态脚本异常不得影响 Canvas、解码或远程输入；非全屏和面板打开状态默认回退为可见；
- 断线清理全屏计时器和本地可见状态，下一次连接从可见工具栏开始；
- 诊断日志继续保存接收、丢弃、完成、提交、显示和解码恢复指标，主界面的简化不删除诊断数据。

## 测试

- Rust 单元测试先验证所有档位仍是 30 FPS、码率层级不变、默认偏好为自动、退化与恢复阈值不回归；
- Bun 单元测试先验证普通窗口始终可见、全屏 3 秒后收起、顶部触发/焦点/面板固定显示、活动时间回退和断线重置；
- Bun 生产构建验证 TypeScript 和 CSS；
- 完整 Rust workspace、Clippy `-D warnings`、格式、Python 发布测试与 Windows 安装器门禁保持不变；
- 两台真实 Windows 电脑上的最终观察由用户手动完成，重点比较远程视频播放、窗口拖动、全屏顶部内容和弱网自动降档。

## 成功标准

1. 新连接默认显示“自动（清晰）”，无需用户手动设置。
2. 自动降到均衡或流畅后仍保持 30 FPS 配置。
3. 主工具栏不再暴露帧计数和累计丢包率。
4. 全屏闲置 3 秒后远程画面无遮挡，鼠标到顶部立即恢复全部控制。
5. 文件/文字面板打开、键盘聚焦或非全屏时工具栏绝不自动隐藏。
6. 不修改协议版本，不要求重新部署公网中继。
