# Windows 0.1.48：文件续传预检

0.1.48 加固 0.1.47 的显式断点续传。目标是让无法安全继续的任务尽早失败，并给用户可以直接采取行动的中文原因。

## 断点内容证明

- 接收端在 `FileDecision` 中返回已暂存前缀的 BLAKE2s 哈希；
- 该证明位于 Noise 端到端加密的文件通道中，中继不能读取；
- 发送端在发送任何剩余数据前重新读取源文件前缀并比较哈希；
- 即使源文件被替换为名称和大小完全相同的文件，也会立即返回“源文件变化”，不会等待最终完整哈希才失败；
- 最终文件仍必须通过原有的完整大小和完整 BLAKE2s 校验。

## 磁盘空间预检

- Windows 接收端使用当前用户实际可用的下载卷空间；
- 所需空间按“剩余文件字节 + 64 MB 安全余量”计算；
- 空间不足时返回独立结果，不再混入普通读写失败；
- 已有断点会继续保留，用户清理磁盘并重试后仍可续传；新建的零字节暂存会立即移除。

## 快速失败

被控端选择并发送文件时，如果源文件读取、断点证明或发送过程失败，控制端会立即结束对应接收任务并显示原因，不再依赖 60 秒空闲超时兜底。

## 协议与兼容性

`FileDecision` 新增可选前缀哈希，`TransferResult` 新增“空间不足”和“源文件变化”，应用协议从 7 升到 8。生产 relay 只匹配协议版本并转发密文，不需要为本次改动重新部署；两台 Windows 电脑必须同时升级到 0.1.48。
