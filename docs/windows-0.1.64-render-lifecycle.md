# Windows 0.1.64：远程画面渲染生命周期

## 问题

控制端收到连接状态事件时，标题和说明文字的变化也会触发整个控制页面重建。远程页面重建会释放并重新创建 WebView2 的 H.264 解码器、Canvas、输入捕获和尺寸观察器；稳定连接中这类无关更新会造成短暂黑屏、首帧重新等待或可感知卡顿。

## 处理

- 将连接状态的“表面身份”限定为连接状态和视频流 ID。
- 连接仍处于同一状态、同一视频流时，只就地更新状态徽标文字和状态样式。
- 只有连接状态变化或视频流 ID 变化才允许重新渲染远程表面。
- 保持原有状态文案、断线清理、输入释放和视频配置切换逻辑；不改变协议 9、加密边界或中继。

## 验证

- 新增 3 项连接表面生命周期单测，覆盖稳定连接、状态变化和视频流变化。
- UI 单测、TypeScript 检查、生产构建、Rust 格式、检查和 Clippy 继续执行。
- 两台 Windows 的实际帧率、输入延迟和长稳仍需手动测试；本次改动不替代物理设备验收。

## 兼容性

- Windows 产品版本：0.1.64；
- 协议：9；
- 公网中继：无需升级；
- macOS：未修改；
- 安装器：未签名 x64 测试版本。
