# DeskLink 0.1.25 Windows 双机验收

这份清单只包含必须由两台真实 Windows 电脑完成的体验验收。不要在记录中填写访问密码、设备身份文件或屏幕中的隐私内容。

## 准备

- 两台电脑都安装 `DeskLinkSetup-0.1.25-x64.exe`，在“关于”确认版本均为 0.1.25。
- 被控电脑保持 DeskLink 打开且显示可被连接；控制电脑能看到“连接另一台电脑”。
- 建议测试期间在两台电脑的“设置”中开启“共享脱敏诊断”，异常会自动关联上传；完成后可以关闭。
- 先使用同一网络，再让其中一台切换到手机热点，覆盖不同公网网络。

## 核心连接

1. 使用临时密码连接，被控端必须出现确认窗口；拒绝后控制端应明确结束，不能自动获得控制。
2. 再次连接并批准，画面、鼠标和键盘均应可用；目标主机状态不能因连接动作变成“服务已停止”。
3. 在控制端取消“正在查找设备”，按钮应立即恢复可用；随后重新连接应成功。
4. 设置固定密码并连接一次；关闭控制端后从历史设备直接连接，设备 ID 与密码应已在当前 Windows 账户下安全保存。
5. 被控端撤销批准或修改密码后，旧记录必须失败且给出可恢复的中文提示。

## 画面与输入

1. 在 100%、125% 和 150% Windows 缩放下，分别点击画面四角、中心和窗口边缘，远端鼠标位置应一致。
2. 拖动窗口、滚动长页面、连续移动鼠标 2 分钟，不应卡死、黑屏或触发“此中继会话正在使用中”。
3. 测试单击、双击、左右键、滚轮、拖拽、常用字母数字和组合键；`Ctrl+Alt+Delete` 仍按界面说明在主机本地操作。
4. 使用“发送文字”输入一段中文，不能因本地输入法组合状态导致按键残留。
5. 全屏与退出全屏各执行三次，工具栏和远程画面不得错位或出现多余滚动条。

## 双显示器

1. 被控端连接两个显示器，确认控制端能看到显示器选择入口和正确的数量、分辨率。
2. 在显示器 1 和显示器 2 之间来回切换五次，每次画面都应刷新到所选屏幕。
3. 每个屏幕都重复四角、中心、拖拽和滚轮测试，鼠标不得落到另一块屏幕或产生固定偏移。
4. 测试两块屏幕左右排列、主屏不在最左侧，以及不同缩放比例；记录具体布局和失败位置。

## 断线恢复

1. 控制中断开控制端网络 10 秒后恢复，只能出现一个恢复流程；被控端保持可连接，不能在“已在线/已中断”之间持续闪烁。
2. 强制关闭控制端，等待 5 秒后重新打开并从历史设备连接；旧连接应被同一控制端安全替换，不能报告“已有另一台连接”。
3. 被控端休眠后唤醒，再连接一次；如果需要重新批准，提示必须明确且不能保留半连接状态。
4. 连续控制 30 分钟，至少包含 10 分钟持续鼠标移动和滚动；画面不得停止更新，输入延迟不应持续增长。

## 通过标准与记录

- 上述项目全部通过，且两种网络环境都能完成“查找、批准、看见画面、输入、断开、重连”。
- 任何失败只记录：发生时间、两台电脑的 DeskLink 状态文字、网络类型、显示器布局、操作步骤和截图；不要记录密码。
- 如果开启了诊断，请额外记录“立即发送诊断”显示的成功时间，便于在云端按时间定位同一会话。
