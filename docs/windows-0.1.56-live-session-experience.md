# Windows 0.1.56：实时会话体验

DeskLink 0.1.56 聚焦远程控制已经建立后的流畅度与全屏操作。它不改变设备身份、端到端加密、中继协议、主机批准、输入、剪贴板或文件传输通道。

## 本次变化

- 新连接默认使用“自动”画质，主机根据现有加密反馈在流畅、均衡和清晰档之间调整；
- 三个画质档都保持 30 FPS，通过分辨率和码率区分档位，弱网降档时优先保留操作连续性；
- 用户仍可在当前会话中手动选择画质，主机确认实际档位后再更新界面；
- 远控工具栏只显示分辨率、当前画质和“已加密”，不再用帧计数与累计丢包率干扰主要操作；
- Windows 原生全屏中的工具栏显示 3 秒后自动收起，鼠标移动到顶部、键盘焦点进入工具栏或打开文字/传输面板时立即显示；
- 进入全屏时短暂提示“鼠标移到顶部显示工具栏 · Esc 退出全屏”，退出全屏或断线会清理计时状态。

## 安全与兼容边界

- 自动画质继续使用端到端加密控制通道，中继无法读取画质反馈或业务内容；
- 没有增加新的网络消息或部署要求，现有 0.1.x 中继无需升级；
- 工具栏可见性只更新本地 DOM 属性，不触发页面重建、视频解码器重建或远程输入状态变更；
- 非全屏、工具栏获得键盘焦点以及文字/传输面板打开时，工具栏始终保持可见。

## 自动验证

- Rust 单元测试覆盖默认自动画质及三个档位的 30 FPS 配置；
- Bun 单元测试覆盖工具栏三秒收起、顶部/焦点/面板固定显示和中文状态摘要；
- Windows 发布门禁继续覆盖前后端版本、中继配置、生产资源、Rust workspace、Clippy、前端生产构建与安装器负载哈希。

两台真实 Windows 电脑上的最终体感由人工确认，重点观察窗口拖动、全屏顶部触发、弱网自动降档和不同 DPI/多显示器组合。
