# DeskLink 两台 Windows 电脑使用说明

0.1.26 发布前的完整人工验收见 [windows-0.1.26-two-pc-acceptance.md](windows-0.1.26-two-pc-acceptance.md)。

## 当前支持范围

同一个 DeskLink Windows 安装包同时包含主机端和控制端。两台 Windows 10/11 x64 电脑安装后，可以由 A 控制 B；如需反向控制，再创建一次反向配对即可。

当前版本支持多显示器切换、H.264 低延迟画面、鼠标、滚轮、常用按键、F1–F12、导航键和中文/符号文字发送。暂不支持整块虚拟桌面合成、文件传输、远程音频和 Windows 安全桌面组合键 `Ctrl+Alt+Delete`。

## 首次连接

### 1. 在被控制电脑上启用远程连接

1. 安装并打开 DeskLink。
2. 在主页面的本机访问面板点击“启用远程连接”。DeskLink 会生成随机会话凭据，使用当前 Windows 用户的 DPAPI 加密保存，并配置官方公网中继。
3. 点击“生成临时密码”，复制页面显示的设备 ID 和临时密码。临时密码短时间有效，只应发送给自己的另一台电脑。

两台电脑可以位于不同网络，不需要开放家庭路由器端口，也不需要允许公网入站。中继只转发经过 Noise 端到端加密的业务数据；QUIC 入口同时使用 Windows 信任的 TLS 证书校验服务器身份。

### 2. 在控制电脑上连接

1. 安装并打开同一个 DeskLink。
2. 进入“控制其他电脑”，输入目标电脑的设备 ID 和临时密码或固定密码。
3. 点击“查找并连接设备”。
4. 回到被控制电脑，核对 Windows 原生确认框中的设备 ID 和公钥指纹，再明确批准。
5. 控制端收到首个视频配置后，才会使用 DPAPI 保存重连材料。以后可以从“已保存设备”直接连接。

## 控制输入

远程画面获得焦点后，可以发送鼠标移动、左右/中键、滚轮、字符键、Enter、Escape、Backspace、Tab、方向键、Delete、Insert、Home、End、Page Up、Page Down、Caps Lock、F1–F12，以及 Shift/Control/Alt/Windows 组合键。

需要输入中文、长文本或系统输入法内容时，使用控制工具栏的“发送文字”，粘贴最多 256 个字符后发送。普通 Windows 进程不能向 UAC/登录安全桌面注入 `Ctrl+Alt+Delete`，此操作仍需在主机本地完成。

## 自建中继（高级）

普通用户不需要修改中继设置。需要完全自主管理基础设施时，可在“设置”的高级连接选项中填写自建中继的 `IP:端口` 与 TLS 服务器名称。两台电脑都必须能够访问该 UDP 地址，且证书链必须受 Windows 信任，证书 SAN 必须与 TLS 服务器名称一致。

服务端主要环境变量：

- `DESKLINK_RELAY_ADDR`：监听地址，例如 `0.0.0.0:4433`；
- `DESKLINK_RELAY_CERT_PEM`：完整证书链 PEM；
- `DESKLINK_RELAY_KEY_PEM`：对应私钥 PEM；
- `DESKLINK_RELAY_SESSION_TTL_S`：可选，会话存活时间。
- `DESKLINK_RELAY_MAX_CONNECTIONS`：可选，同时 QUIC 连接上限；
- `DESKLINK_RELAY_MAX_SESSIONS`：可选，同时会话上限。

未提供 PEM 时生成的 `localhost` 自签名证书只适用于自动化测试和本机开发，不能作为普通用户的生产连接方式。

## 诊断与恢复

“设置”中的双机连接诊断会区分配置状态、可信设备存储、中继/安全会话运行状态和主机运行状态：

- `DL-CFG-101`：连接设置能否由当前 Windows 用户安全读取；
- `DL-SEC-102`：已批准设备存储与本地审批边界是否可用；
- `DL-NET-201`：中继与安全会话当前处于已连接、重连、等待或停止状态；
- `DL-HOST-301`：主机运行组件是否正常。

“导出诊断报告”会在当前用户的“下载”文件夹生成中文纯文本报告。报告不会包含会话 ID、中继密钥、设备公钥、完整设备身份或访问密码；发送前仍应自行检查内容。

常见问题：

- “查找并连接设备”失败：先确认两台电脑均能正常访问互联网，并检查设备 ID 和访问密码。
- 一直显示“请在主机上批准”：检查主机是否显示原生确认框，或临时密码是否已经过期。
- 已批准但没有画面：更新 Microsoft Edge WebView2 Runtime，并更新显卡驱动。
- 重连失败：如果主机在高级设置中重新生成了会话凭据，需要重新配对。
- 下载或启动提示“不安全”：这是 Authenticode/SmartScreen 发布信誉问题，不是 Windows 兼容模式。正式分发必须按 [`windows-code-signing.md`](windows-code-signing.md) 签名应用和安装器。

## 仍需发布验收的项目

自动化测试不能替代两台物理电脑上的真实网络与睡眠行为。正式发布前仍需完成数小时双机运行、断网恢复、Wi-Fi 切换、实际睡眠/唤醒、干净 Windows 用户安装/升级/卸载和已签名安装包 SmartScreen 验收。
