# Windows 0.1.46：传输超时恢复

## 用户结果

- 远端长期不处理文件确认窗口时，控制端不会永久停留在“等待确认”；
- 远端文件选择、文件接收停滞和发送后的保存确认都有明确截止时间，超时后可以直接重试；
- 一项剪贴板操作进行期间不能重复提交，15 秒无响应会恢复按钮；
- 迟到、重复或属于旧操作的剪贴板和文件响应不会覆盖当前状态，也不会触发远程粘贴；
- 超时只取消对应的剪贴板或文件操作，远程画面、输入和系统声音继续运行。

## 截止时间

| 阶段 | 截止时间 | 超时结果 |
|---|---:|---|
| 剪贴板发送、读取或智能粘贴 | 15 秒 | 显示失败并允许重新操作 |
| 等待远端确认接收文件 | 2 分钟 | 取消本次发送并暂停后续队列 |
| 等待远端选择文件 | 5 分钟 | 发送取消请求并提供“重新获取” |
| 已开始接收但没有新分片 | 60 秒 | 删除临时文件并允许重新获取 |
| 单个文件分片无法写入可靠通道 | 30 秒 | 停止本次发送并保留重试入口 |
| 文件发送完成但没有保存确认 | 60 秒 | 停止等待，提示先检查远端下载目录 |

文件队列在失败后保持暂停，避免无人值守时连续弹出多个确认窗口。用户确认环境恢复后可以手动继续队列。

## 请求关联与协议

剪贴板响应必须同时匹配请求 ID 和操作类型。远端文件选择返回的 `FileOffer` 新增可选 `request_id`：直接上传使用 `None`，响应“从远端获取文件”时必须携带原请求 ID。控制端只接受当前仍在等待的精确请求，其他邀请默认拒绝。

该消息布局把应用协议从 5 升到 6。生产 relay 只比较并转发协议版本与密文，不需要重新部署；两台 Windows 电脑需要升级到相同的 0.1.46。
