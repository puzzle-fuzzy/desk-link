# Windows 0.1.65：瞬时重连画面连续性

## 问题

控制端在已有远程画面后遇到可恢复的中继或主机连接故障时，会先清空视频配置、关闭解码器并回到连接进度页。短暂网络抖动因此表现为黑屏、重新等待首帧和输入体验中断，即使后台连接很快能够恢复。

## 处理

- 仅当已有视频配置和活动控制通道，并且运行状态从已连接进入重连时，保留 Canvas、解码器和最后一帧。
- 重连等待期间释放按键/鼠标状态，阻止迟到输入；不向主机注入旧操作。
- 远程画面显示“连接暂时中断，画面保持不变”覆盖层和重试说明；声音状态提示连接恢复后继续。
- 首次连接尚未收到视频时仍使用原有连接进度页；永久停止、主动取消和新视频流仍完整清理旧表面。
- 协议 9、Noise 会话、中继和文件/剪贴板协议均不改变。

## 验证

- 新增重连表面保留边界单测，覆盖已连接重连和首帧前重连。
- 前端全量测试、TypeScript、Rust 格式/检查/Clippy、发布构建和安装器嵌入校验继续执行。
- 两台 Windows 的实际网络抖动、帧连续性和输入恢复仍需人工测试。

## 兼容性

- Windows 产品版本：0.1.65；
- 协议：9；
- 公网中继：无需升级；
- macOS：未修改；
- 安装器：未签名 x64 测试版本。
