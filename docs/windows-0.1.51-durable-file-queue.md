# Windows 0.1.51：等待发送队列跨重启恢复

0.1.51 将 0.1.50 的单文件任务恢复扩展到多文件等待队列，解决应用退出、连接切换或电脑重启后尚未开始的文件静默丢失问题。

## 恢复行为

- 最多 20 个等待文件使用当前 Windows 账户的 DPAPI 加密保存，并通过原子替换写入 `file-queue-recovery.bin`；
- 恢复记录绑定原目标的完整 128 位设备身份，WebView 只获得 12 位公开设备 ID、文件名和大小；
- 连接原设备时，恢复队列只以暂停状态载入，用户必须明确点击“继续队列”才会开始发送；
- 连接其他设备时，后端拒绝添加或发送新文件，界面明确提示旧队列归属，并允许用户选择“放弃旧队列”；
- 清空或放弃队列只删除加密元数据，不删除任何本机源文件；陈旧的界面 revision 不能清理更新后的队列；
- 恢复记录超过 24 小时、目标身份无效、任务重复、数量越界或密文损坏时失败关闭并安全清理。

## 源文件变化

每个排队文件同时记录名称、大小和最后修改时间。真正发送前会重新读取文件元数据：

- 文件不存在、权限不可用、名称、大小或修改时间变化时，该项不会发送；
- 无效项会从队列移除；若后面还有文件，队列保持暂停并提示用户重新添加；
- 传输 ID 在安全重验后保持不变，避免恢复过程生成语义不同的任务。

## 协议与部署

队列恢复只修改控制端本地加密状态和 Windows 界面，不改变 DeskLink 协议或线上 relay。协议版本保持 8，中继无需重新部署。
