# Windows 0.1.66 远程画面生命周期

0.1.66 将控制端的远程画面表面状态收敛为三个纯状态：`empty`、`live`、`retaining`。

- `live` 表示会话已经建立；首帧可以晚于连接状态到达。
- `retaining` 表示网络抖动或重试握手期间继续复用最后一帧、Canvas 和解码器。
- `empty` 表示首次连接尚未产生画面，或会话已停止、取消、切换到新目标。

状态机只接受运行时状态、上一个表面状态和“是否存在可复用视频表面”三个输入，不直接操作 DOM 或 WebCodecs。这样可以稳定覆盖完整重试路径，并防止后续修改状态标题或重试文案时重新拆掉远程画面。

本版本不改变协议、中继或业务数据格式；两台 Windows 电脑之间的真实长时间抖动测试仍需在目标环境手动完成。
