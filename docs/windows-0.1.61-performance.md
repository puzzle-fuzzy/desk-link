# Windows 0.1.61：极致性能阶段

## 优化目标

本阶段优先降低控制端本机延迟和高频事件 CPU 开销，保持协议 9、视频邮箱容量、输入顺序和远程安全边界不变。优化针对真实热路径，而不是牺牲可靠性换取未经测量的微小收益。

## 已完成优化

- 控制端状态驱动的 UI 更新使用 `requestAnimationFrame` 合帧，同一任务内的连续信号只提交一次界面更新。
- 鼠标输入队列从 `Array.shift()` 改为头指针队列，避免队列增长时搬移全部元素；离散按键、按钮和释放顺序保持不变。
- 远程游标、画布和视口节点在视频会话建立时缓存，游标事件不再重复执行 DOM 查询。
- Canvas 2D 上下文开启 `desynchronized` 低延迟提示；视频帧尺寸与画布一致时使用直接绘制路径，避免无意义缩放。
- 首帧等待提示只在首帧隐藏，不再在每一个视频帧上查询 DOM。

## 基线与验证

0.1.60 构建时前端生产包为 JS `164,853` 字节、CSS `69,700` 字节；0.1.61 构建后 JS 为 `165,890` 字节、CSS 仍为 `69,700` 字节。新增的调度器只有少量代码开销，换取高频状态更新时避免重复整页重建；本阶段不引入新的运行时依赖。Windows UI 测试由 `117` 项增加为 `120` 项，新增渲染合帧测试 3 项，输入队列既有行为全部通过，TypeScript 检查和生产构建通过。

真实帧率、CPU、GPU 和输入端到端延迟仍需在两台 Windows 电脑上用同一远程会话手动测量；自动测试无法替代 WebView2、硬件解码器和不同显示器缩放组合的实际数据。

- macOS：本阶段未修改；
- 诊断服务：不需要新增字段，继续使用已部署的 0.1.60 诊断服务；
- 安装包：仍为未签名测试版本。
