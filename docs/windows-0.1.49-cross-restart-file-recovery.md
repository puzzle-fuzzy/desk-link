# Windows 0.1.49：跨重启文件传输恢复

0.1.49 补齐文件断点续传在应用退出或电脑重启后的恢复入口。它不自动恢复业务操作，而是安全保留用户下一次可以主动重试所需的最小状态。

## 恢复行为

- 文件发送、远端文件获取和已有断点接收开始时，记录最近一次可重试操作；
- 网络断线、远端拒绝、超时或用户取消后保留恢复记录；
- 下一次启动会显示中文恢复提示，连接设备后自动展开“剪贴板与文件”面板；
- 用户必须点击“重新发送”或“重新获取”，DeskLink 不会在后台静默传输；
- 接收端明确确认文件已保存后立即清理恢复记录。

## 本地安全边界

- 恢复记录位于当前 Windows 账户的 Local AppData，仅使用当前账户 DPAPI 加密；
- 写入采用临时文件与原子替换，避免断电留下半份 JSON；
- 发送源路径只存在于 Rust 和 DPAPI 密文中，WebView 只获得文件名、方向、大小和中文提示；
- 记录在 24 小时后自动失效，损坏、被替换或属于其他账户的密文会失败关闭；
- 无法读写恢复记录不会阻止当前文件传输，只会提示本次运行结束后可能无法恢复。

## 续传校验

发送任务重试时会重新读取源文件，并检查名称、大小、传输 ID 和接收端已有前缀哈希。接收任务继续沿用同卷暂存、剩余空间预检和最终完整哈希校验，因此跨重启恢复不会降低 0.1.48 的内容一致性要求。

## 协议与部署

本次只增加 Windows 本机恢复状态，不修改线上中继或端到端协议；协议版本保持 8，生产 relay 不需要重新部署。两台 Windows 电脑仍建议安装同一个最新版本。
