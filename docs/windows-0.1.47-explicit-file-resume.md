# Windows 0.1.47：显式文件断点续传

这一阶段解决文件传输因网络中断后只能从头开始的问题。DeskLink 不会在后台静默重发；用户需要在连接恢复后点击重试，被控端也仍需再次确认接收或重新选择文件。

## 行为

- 接收端按传输 ID 把未完成数据保存在“下载”文件夹同卷的隐藏暂存目录；
- 连接意外中断时同步并保留已有数据，显式取消、空闲超时、无效分块和完整哈希失败仍立即删除；
- 重试复用原传输 ID，接收端重新读取暂存前缀、恢复 BLAKE2s 状态并返回准确字节偏移；
- 发送端仍从头读取并计算完整文件哈希，但只发送偏移量之后的数据，偏移量落在分块中间时也会精确裁剪；
- 从远端获取文件时，远端用户必须重新选择文件；只有名称与大小匹配才复用断点 ID，否则作为新文件发送；
- 完整文件只有在大小和端到端哈希都通过后才移动到下载目录，且不会覆盖已有同名文件；
- 未完成暂存超过 24 小时后会由后续接收操作清理。

## 协议与兼容性

`FileSelectionRequest` 新增可选续传提示，`FileDecision` 新增接收偏移量，应用协议从 6 升到 7。生产 relay 只匹配协议版本并转发密文，不需要为本次改动重新部署；参与连接的两台 Windows 电脑必须同时升级到 0.1.47。

## 验证重点

- 暂存文件恢复、跨分块偏移裁剪、完整哈希与同名文件保护；
- 非零请求 ID、续传元数据、失败决定零偏移和最大文件边界的协议校验；
- 控制端发送、远端发送两条方向都使用同一续传语义；
- 取消和接收空闲超时不会留下可被误恢复的数据。
