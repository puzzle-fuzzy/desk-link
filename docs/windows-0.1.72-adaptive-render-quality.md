# Windows 0.1.72 自动画质与本地显示压力

0.1.71 已经能记录控制端 Canvas 因解码输出过快而合并的显示帧。0.1.72 开始把这个信号接入已有的自动画质闭环，让“看得见的卡顿”可以在同一会话内自我缓解。

## 处理路径

1. 解码器输出回调在替换待显示帧时只增加两个内存计数器，不在视频热路径创建对象或发起 IPC。
2. 控制端每秒取出一次周期计数；低于 8 次的偶发合并帧直接忽略。
3. 一秒内达到 8 次时，上报一个已有协议 9 的播放压力样本；达到 24 次时上报更高的压力提示。字段仍是 `decode_queue_peak`，中继不会解析或转发业务明文。
4. 主机只在“自动画质”下处理反馈，并继续使用原有的连续样本、降档冷却和健康窗口。因此短暂的桌面合成抖动不会立刻改变画质，持续压力才会逐步从清晰降到均衡或流畅。

## 边界

- 用户手动选择流畅、均衡或清晰时，主机忽略自动反馈，不会偷偷改动用户设置。
- 诊断中的累计显示合并帧不会被周期上报清零，导出的报告仍能看到完整会话数据。
- 上报失败只丢弃这一秒的自适应提示，不影响视频、输入、音频、剪贴板或文件传输；下一秒会重新采样。
- 这是发送端负载调整，不是本地 GPU 加速开关。若降到流畅后仍持续卡顿，应继续查看显示帧率、最大帧间隔和 WebView2/显卡环境。

## 验证

- TypeScript 单测覆盖偶发、持续、严重和非法显示合并帧样本。
- 既有主机 `AdaptiveVideoQuality` 单测继续验证连续压力才降档、手动档位不受影响，以及恢复需要健康窗口。
- 发布前仍需在两台真实 Windows 电脑上分别验证自动画质和手动画质，确认跨网、窗口遮挡和双显示器场景下没有频繁抖动。
