# Windows 0.1.37：远程画面缩放

## 目标

控制端工具栏新增“适应 / 1:1”缩放选择，解决高分辨率和多显示器场景下“既要看完整画面，也要检查原始细节”的切换需求。此功能只改变本机如何显示已收到的远程视频，不修改视频内容、加密协议或中继行为。

## 用户行为

- **适应**：保持完整远程画面可见，并按当前窗口等比例缩放。这是首次使用和异常偏好值的默认状态。
- **1:1**：一个收到的视频像素对应一个界面像素。画面超过窗口时自动居中，并提供横向和纵向滚动条。
- 选择结果保存在当前 Windows WebView 用户数据中，下次打开控制会话继续使用。
- 切换显示器或重建视频解码器不会丢失缩放选择。

## 坐标安全

鼠标输入始终按画布当前实际边界计算，不按远程分辨率或窗口外框猜测。1:1 模式滚动后，远程光标位置会把滚动量转换为滚动内容坐标，避免重复扣除滚动距离。滚动事件也会刷新缓存边界，因此缩放、滚动和高 DPI 小数边界使用同一条坐标链。

## 兼容性

缩放属于控制端本地展示能力，没有提升协议版本，也不需要重新部署公网中继。0.1.37 控制端可连接当前协议版本相同的 0.1.36 及以上被控端；为了减少排查变量，双机测试仍建议安装同一最新版。

## 自动验证

- 缩放偏好缺失、异常和存储不可用时安全回退到“适应”；
- 1:1 模式偏好可保存和恢复；
- 画布留黑边、高 DPI 小数边界与滚动后的远程光标坐标均有单元测试；
- Windows UI TypeScript 检查与 Vite 生产构建通过后再进入 Rust 发布验证和安装器封装。
