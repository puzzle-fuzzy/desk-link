# Windows 0.1.75 全屏工具栏指针热路径

全屏状态下，控制端会监听文档级 `pointermove`，用来在鼠标靠近顶部或工具栏时显示工具栏。旧路径每次事件都重新查找工具栏节点并匹配 `:hover`，会把本来只需要判断事件归属的操作变成选择器和样式命中工作。

## 新路径

- 绑定控制端交互时缓存当前工具栏元素；远程画面重建时清空缓存。
- 全局指针移动只检查 `event.target` 是否属于缓存工具栏节点，其他逻辑保持不变。
- 工具栏自身的 `pointerenter`、`pointerleave` 和焦点事件仍负责更新可见性状态，因此不会改变显示行为。

## 验证

- TypeScript 生产构建和完整前端单测通过。
- 真实测试重点观察全屏顶部呼出、工具栏内部移动、离开工具栏后自动收起，以及远程画面鼠标控制是否不受影响。
