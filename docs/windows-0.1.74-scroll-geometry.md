# Windows 0.1.74 滚动坐标热路径

在“1:1 浏览”模式下，远程画布可能大于本地窗口。旧路径每次滚动都调用 `getBoundingClientRect()`，而滚动事件又可能和 Canvas 绘制、光标更新处于同一批 WebView2 任务中，容易产生同步布局读取。

## 新路径

- 初始布局、窗口尺寸变化和画布尺寸变化仍通过一次真实布局读取建立缓存。
- 滚动时只根据 `previousScroll - nextScroll` 平移缓存的画布边界，宽高和视口边界保持不变。
- `pointerdown` 仍会主动刷新真实边界，确保开始拖动或点击时使用最新几何信息。
- 切换“适应/1:1”、重连和远程画面重建会重置滚动基线，避免不同会话之间复用旧坐标。

## 验证

- 单测验证滚动偏移能把缓存画布边界从视口坐标正确平移到新位置。
- 既有坐标映射、双显示器和 1:1 本地平移单测继续通过。
- 真实测试重点观察 1:1 滚动后鼠标点击位置、远端光标位置和切换显示器后的坐标一致性。
