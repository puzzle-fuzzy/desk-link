# Windows 0.1.89：清晰档与高 DPI 缩放

## 调整内容

- 清晰档平均码率从 12 Mbps 提高到 18 Mbps，峰值预算从平均值的 1.25 倍提高到 1.5 倍；窗口移动、滚动和小字重绘时有更多瞬时码率空间。
- Media Foundation 清晰档的质量/速度目标提高到 95，仍保留低延迟、CABAC 和峰值约束 VBR；不支持这些可选属性的硬件编码器会安全忽略。
- BGRA/RGBA 转 NV12 在源分辨率高于编码分辨率时改用双线性重采样。原先的最近邻取样会在 4K/高 DPI 缩放时丢失细字覆盖率，新的路径只在发生缩放时启用，1:1 仍保持快速路径。
- 自动画质的降档机制没有改变协议或中继，只在网络丢包、解码积压和显示压力持续升高时保护交互可用性。

## 预期效果

- 目标电脑为 2K/高 DPI 屏幕时，细字体、窗口边框和浅色线条的锯齿与块状模糊减少。
- 同一网络下选择“清晰”时，画面压缩纹理减少；网络不足时，自动模式仍会回退到均衡或流畅，避免输入延迟无限累积。

## 手动验收

1. 两台 Windows 都安装 0.1.89，在控制端选择“清晰”。
2. 目标电脑打开包含小字号中文、表格细线和滚动列表的窗口，比较适应模式与 1:1 原始像素模式。
3. 观察连接状态中的分辨率与画质摘要；若网络持续承压，自动模式应降档但保持鼠标和键盘可用。
