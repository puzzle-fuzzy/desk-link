# Windows 0.1.31：双向文件传输

DeskLink 的文件传输现在支持两个方向：

- 控制端选择本机文件并发送到被控端；
- 控制端请求被控端选择文件，再接收到控制端本机。

## 用户流程

远程控制连接成功后，打开“传输”面板：

1. “发送文件”会在控制端打开 Windows 文件选择器，被控端确认后接收；
2. “获取远端文件”会在被控端打开 Windows 文件选择器，被控端选择后立即回传；
3. 接收完成的文件保存在当前 Windows 账户的“下载”文件夹；
4. 同名文件不会被覆盖，DeskLink 会自动添加数字后缀；
5. 传输可以取消，失败或取消后可以重新发送或重新获取。

## 安全与稳定性

- 文件内容继续使用现有端到端加密会话；
- 文件数据使用独立可靠通道，不与鼠标、键盘、视频和音频队列混用；
- 每个文件最大 256 MB，分块最大 48 KB；
- 接收端按顺序写入临时文件，并在完整大小和 Blake2s 哈希校验通过后发布；
- 断线、取消或校验失败时会删除未完成的临时文件；
- 同一会话同时只运行一个文件任务，避免两个方向互相抢占。

该功能只修改 Windows 客户端和端到端传输协议，不需要重新部署公网中继。
