# Windows 0.1.62：会话生命周期加固

## 本阶段目标

性能实测由两台 Windows 电脑手动完成；代码侧先收紧远程控制的失败边界，避免主机任务异常退出后界面仍停留在旧状态，或控制端在会话已经结束后继续接受迟到的输入、剪贴板和文件命令。

## 已完成

- 主机 supervisor 发生不可恢复错误、重连预算耗尽或会话失效时，UI 和托盘都会收到明确的 `Stopped` 生命周期事件，不再停留在“连接中/正在重连”的旧状态。
- 控制端后端在 `idle` 或 `stopped` 状态拒绝新的控制、剪贴板、文件和画面控制命令；迟到的 WebView 调用会得到可操作的中文错误，不会进入已结束任务的命令队列。
- 保持现有协议 9、端到端加密、双屏切换、视频邮箱、音频、剪贴板和文件恢复格式不变。

## 验证

- Rust 控制端新增“运行状态已停止后拒绝命令”单测。
- 主机生命周期仍使用既有停止原因脱敏逻辑，不向 UI 暴露本地路径或内部密钥材料。
- 两台 Windows 的长稳、断网、睡眠/唤醒和实际输入延迟仍由用户手动实测；本阶段不把离线单测当作硬件验收。
