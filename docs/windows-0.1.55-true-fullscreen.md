# Windows 0.1.55：原生全屏远程桌面

0.1.55 修复控制端点击“全屏”后仍保留应用窗口边界、远程画面没有真正占满显示器的问题。

## 根因

旧实现调用 WebView 的页面 Fullscreen API，只能把 `.remote-session` 放大到 WebView 可用区域，无法可靠控制 Tauri 窗口本身和 Windows 任务栏。与此同时，“适应”模式中的 Canvas 使用固有尺寸配合最大宽高，只会在窗口较小时缩小，不会在控制端显示器更大时主动放大。

## 新行为

- 使用 Tauri 窗口 API 进入 Windows 原生全屏，并只开放读取与设置全屏所需的最小窗口权限；
- 全屏时隐藏 DeskLink 自定义标题栏、普通导航、页面标题和无关提示；
- 远程画面固定覆盖整个物理显示区域，工具栏不再占用视频布局高度，而是作为顶部覆盖层保留；
- 文件传输面板和发送文字输入框在全屏中显示为可操作的覆盖面板；
- “适应”模式会向上或向下缩放远程画面，同时保持原始宽高比，不裁剪、不拉伸；
- 鼠标使用缩放后真实画面矩形映射，宽高比不同产生的黑边不会造成位置偏差；
- `Esc`、工具栏按钮、主动断开和意外断线都可以安全退出全屏；切换全屏前释放残留输入，避免在远端留下按下状态。

## 验证边界

自动测试覆盖 16:10 控制端显示 16:9 远端画面、横屏控制竖屏以及无效源尺寸，确认 contain 画面矩形和输入边界正确。前端测试和生产构建会验证 Lucide 图标、中文界面与 TypeScript 类型。Windows 安装包仍为未签名测试版本；原生全屏的任务栏覆盖、不同 DPI 组合和多显示器所在屏幕需由两台实际 Windows 电脑完成最终体感验证。

DeskLink 协议仍为版本 8，本次没有改变中继或业务密文格式，公网中继无需重新部署。
